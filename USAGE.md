# Как пользоваться Claude Code

## Основы

Claude Code — это AI-ассистент в терминале. Он читает код, редактирует файлы, выполняет команды и объясняет решения.

### Запуск

```bash
./claude-docker              # интерактивный сеанс
./claude-docker "запрос"     # одноразовый вопрос
./claude-docker --resume     # продолжить последнюю сессию
./claude-docker --continue   # продолжить с последнего места
```

Windows:

```cmd
claude-docker.bat
claude-docker.bat --resume
claude-docker.bat "что делает код?"
```

## Как ставить задачи

Главное правило: **описывай ЧТО сделать, а не КАК**.

### Плохо
```
добавь в файл server.py на строку 42 проверку if x > 0
```

### Хорошо
```
добавь валидацию что x не может быть отрицательным в server.py
```

Клод сам найдёт нужное место и выберет способ реализации.

### Ещё примеры хороших запросов

```
объясни как устроена аутентификация в этом проекте
```
```
найди баг: при нажатии кнопки сохранить ничего не происходит
```
```
добавь логирование всех запросов в API
```
```
напиши тесты для функции calculate_price
```
```
отрефактори этот файл — он слишком длинный
```

## Работа с кодом

### Чтение и анализ
- Клод видит всю кодовую базу — спрашивай про любые файлы
- Может объяснить архитектуру, найти связи между компонентами
- Ищет баги, анализирует зависимости

### Редактирование
- Клод сам редактирует файлы (Edit, Write)
- Показывает diff перед применением (если включены подтверждения)
- Может создавать новые файлы

### Команды
- Клод выполняет Bash-команды внутри контейнера
- Может устанавливать пакеты, запускать тесты, собирать проекты
- Доступны: git, python, npm, go, curl и базовые утилиты

## Контекст проекта

Перед началом работы полезно дать Клоду контекст:

```
я пишу телеграм-бота на Python. В проекте есть файлы bot.py, handlers.py, config.py.
Стек: aiogram, PostgreSQL, Redis.
```

Или просто попроси Клода изучить проект:

```
изучи этот проект и расскажи что ты понял
```

## Советы

### Уточняй, если не то
Если Клод сделал не то — просто скажи:
```
нет, я имел в виду другое. Нужно чтобы валидация была на уровне модели, а не в обработчике
```

### Разбивай большие задачи
Вместо «напиши мне интернет-магазин»:
```
1. Создай структуру проекта и модели данных
2. Добавь API для каталога товаров
3. Реализуй корзину
4. Добавь оформление заказа
```

### Используй флаги
```bash
./claude-docker "запрос"      # быстрый ответ, без интерактива
./claude-docker --resume          # продолжить последний сеанс
./claude-docker --continue        # продолжить с того места где остановился
```

### Файл CLAUDE.md
Создай в проекте файл `CLAUDE.md` — Клод читает его при старте. Туда можно записать:
- Описание проекта
- Соглашения по коду
- Что нельзя трогать
- Команды для сборки и тестов

Пример `projects/my-project/CLAUDE.md`:
```markdown
# My Project

Стек: Python 3.12, FastAPI, SQLAlchemy, PostgreSQL.

## Команды
- Запуск: `uvicorn main:app --reload`
- Тесты: `pytest`
- Линтер: `ruff check .`

## Правила
- Не менять файлы миграций вручную
- Все запросы к БД через репозитории
- Логирование через structlog
```

## Ограничения контейнера

- Контейнер видит только папку `claude_in_box/`
- Нет доступа к Docker-демону хоста
- Нет доступа к сети хоста
- Если нужно больше — смотри README.md, раздел «Как менять права»
